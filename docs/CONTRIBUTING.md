# Contributing

Thanks for considering contributing.

## Principles

TokenScope is:

- zero-config
- local-first
- privacy-first
- macOS-first for MVP

## Before Opening a PR

- Keep the change focused
- Add tests for parser changes
- Add tests for storage, ingestion, pricing, or UI behavior changes
- Avoid adding dependencies unless necessary
- Do not add network calls, telemetry, login, OAuth, or cloud sync
- Update docs when behavior, architecture, or roadmap status changes

## Parser Changes

Parser changes should include sanitized fixtures.

Do not commit real private logs.
